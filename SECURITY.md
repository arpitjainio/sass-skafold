# Security Policy

## Supported scope

This repository is an open-source starter template. Security reports are welcome for:

- vulnerable defaults that affect fresh installations
- exposed secrets or unsafe configuration patterns
- auth, authorization, or data-handling flaws in the starter itself
- dependency or supply-chain risks that materially affect the template

## Reporting a vulnerability

Please do not open a public issue for a suspected security vulnerability.

Instead:

1. Use GitHub's private vulnerability reporting for the repository if it is enabled.
2. If private reporting is not enabled yet, contact the maintainers privately.
3. Include reproduction steps, impact, affected files, and any mitigations you already tested.

## Response expectations

The goal is to acknowledge valid reports quickly, reproduce the issue, and ship a fix or mitigation as soon as practical. Because this is an OSS starter and not a managed hosted product, response times may vary.

## Responsible disclosure

Please give maintainers a reasonable chance to investigate and patch before public disclosure.
